Global attribute

Global attribute คำอธิบาย
accesskey ใช้กำหนดปุ่ม shortcut เพื่อใช้ย้ายหน้าจอไป focus ในหัวข้อของ tag นั้นๆ
class ใช้ระบุชื่อของ class เพื่อใช้งานคุณสมบัติของ class นั้นๆแก่ element ภายใน tag โดยอ้างอิงจาก style sheet (ใน 1 tag สามารถมี class ได้มากกว่า 1, element)
contenteditable ใช้ระบุ value ว่าจะอนุญาตให้มีการแก้ไขเนื้อหาขณะที่เปิดอยู่บน browser ได้หรือไม่ (value= true หรือ false)
data-* ใช้ระบุช้อมูลของข้อมูลโดยที่ในการแสดงข้อมูลไม่ได้ต้องการแสดงข้อมูลของ data-* ออกมา (ยกตัวอย่างเช่น แสดงชื่อของคนสามคน ชื่อ A,BและC ข้อมูลที่แสดงมีเพียงเท่านี้แต่ระบบจะรู้ว่าA,B,C อายุเท่าไหร เพศอะไร สูงเท่าไหรเป็นต้น : data-*คือ อายุ เพศ ส่วนสูง )
dir ใช้บอกลำดับของข้อมูล value มี ltr (left to right) และ rtl (right to left) (ยกตัวอย่าง : ใช้ตั้งค่าให้แสดงผลตามธรรมชาติของภาษาที่ต่างกันจากการอ่านซ้ายไปขวาหรือขวาไปซ้ายเป็นต้น)
draggable ใช้ระบุว่า element ภายใน tag สามารถใช้เม้าส์ลากข้อมูลไปใช้ได้หรือไม่ (อาจใช้คู่ dropzone)
dropzone ใช้ระบุพื้นที่ในเมื่อลากข้อมูล(drag)จะทำการ คัดลอก,เคลื่อนย้าย หรือ เชื่อมต่อ เมื่อปล่อยใส่พื้นที่ dropzone
hidden element ใน tag จะถูกซ้อนในหน้า web page ( ในอีกความหมายเนื้อหาของ element ใน tag อาจจะไม่มีความเกี่ยวข้องกับ Web page แล้ว)
id ใช้ประกาศความเป็น เอกเทศน์ ของ tag (ยกตัวอย่าง : ประกาศ id ของ tag แล้ว สามารถเรียกใช้ id นั้นเพื่อสั่งใช้งานบางอย่าง id ที่ถูกระบุจะได้รับการทำงานเท่านั้น tag ที่ไม่มีid จะไม่ได้รับผลจากคำสั่ง กล่าวอีกอย่างคล้ายกับการตั้งชื่อให้ tag )
lang ใช้ระบุภาษาที่จะใช้ใน element ของ web page
spellcheck ใช้ระบุให้ตรวขไวยากรณ์ของภาษาภายในเนื้อหา element ใน tag หรือไม่
style ใช้งาน CSS แบบเขียนภายในบรรทัด(inline) แก่ element ภายใน tag
tabindex ใช้กำหนดให้ tab เป็น short key ให้ focus หัวข้อของ tag นั้นๆตามลำดับที่กำหนดไว้ (ทำให้ใช้งาน web page นั้นๆโดยไม่ได้ใช้เม้าส์ได้เช่นกัน)
title ใช้ในการให้คำอธิบายเพิ่มเติมกับ element ใน tag (เมื่อใช้เม้าส์ชี้ element บน web page คำอธิบายเพิ่มเติมจะขึ้นมาที่ปลายเม้าส์ขณะชี้)
translate ใช้ระบุว่าเนื้อหาของ element ควรแปลภาษาหรือไม่ value ของ attribute นี้คือ yes หรือ no (แปลเป็นภาษาท้องถิ่น)

(หมายเหตุ : ตารางนี้เฉพาะที่ทำงานโดยตรงต่อ element เป็นหัวข้อไป กล่าวคือไม่ใช่การ applies ลงใน < body> และมีผลโดยรวมทั้ง page)

(หัวข้อนี้ต้องการความรู้เรื่อง java script ,วิธีการทำงาน การอ่าน code (เขียนแบบไ่ม่รู้พวกนี้ไปก่อน))

Event attribute value คำอธิบาย
onblur script ทำให้เกิด event เมื่อเป้าของการ focus ถูกนำออกไป (กล่าวคือ สมมุติ code ให้ช่อง input พื้นหลังกลายเป็นสีชมพูเมื่อ focus และ ให้พื้นหลังเป็นตัวแปรว่างที่ blur ผล : เมื่อคลิกที่ช่อง input พื้นหลังจะเป็นสีชมพูและเมื่อคลิกที่อื่นพื้นหลังจะกลายเป็นสีขาว) *อ้างอิง https://developer.mozilla.org/en-US/docs/Web/API/Element/blur_event
onchange script แสดงเหตุการณ์เมื่อ value ของ element ถูกเปลี่ยนแปลงค่า
onfocus script เมื่อคลิก, short key หรือ เหตุการณ์ใดๆ ไปยังหัวข้อที่มีตัวแปร focus จะเข้าสู่สถานะ focus และทำให้เกิด event ตาม script ใน focus
oninput script script event จะถูกทำงานเมื่อผู้ใช้งาน input ข้อมูล
oninvalid script script event จะถูกทำงานเมื่อ element เข้าสู่สถานะ invalid
onreset script script event จะถูกทำงานเมื่อปุ่ม reset ของ form ถูกคลิก
onsearch script script event จะถูกทำงานเมื่อผู้ใช้งาน้ขียนบางอย่างบนช่อง search field (< input="search">)
onselect script event ทำงานเมื่อหลังจาก text ถูกเลือกภายใน element
onsubmit script ทำงานเมื่อเกิดการ submit ข้อมูลภายใน form

oncopy script script จะทำงานเมื่อผู้ใช้งานทำการ คัดลอก(copy) เนื้อหาภายใน element
oncut script script จะทำงานเมื่อผู้ใช้งานทำการ คัดลอกแบบตัด(cut) เนื้อหาภายใน element
onpaste script script จะทำงานเมื่อผู้ใช้งานทำการ วาง(paste) เนื้อหาภายใน element

Media event

ทำงานร่วมกับ media ต่างๆและ img (ข้อสงสัย img ใช้ในหัวข้อนีแบบไหน)

Media events attribute value คำอธิบาย
onabort script script จะถูก run เมื่อ abort ทำงานไม่ถูกต้อง onabort จะถูก run แทน
oncanplay script script จะถูก run เมื่อไฟล์พร้อมที่จะเล่น (เมื่อไฟล์ถูกโหลดจนสามารถเล่นได้)
oncanplaythrough script script จะถูก run เมื่อไฟล์พร้อมที่จะเล่นโดยไม่มีการหยุด pause หรือ โหลดไฟล์ (buffering)
oncuechange script script จะถูก run เมื่อ cue ถูกเปลี่ยนใน element ของ
ondurationchange script script จะถูก run เมื่อความยาวของ media ถูกเปลี่ยน
onemptied script script จะถูก run เมื่อบางอย่างผิดปกติเกิดขึ้น เช่น การขาดการเชื่อมต่ออย่างคาดไม่ถึง
onended script script จะถูก run เมื่อ media เล่นไปจนถึงจุดจบของ media
onerror script script จะถูก run เมื่อการโหลดไฟล์เกิดการ error
onloadeddata script script จะถูก run เมื่อ media ถูกโหลด
onloadedmetadata script script จะถูก run เมื่อ meta data (เช่นความยาวของ media เป็นต้น) ของ media ถูกโหลด
onloadstart script script จะถูก run เมื่อไฟล์กำลังเริ่มกระบวนการโหลด
onpause script script จะถูก run เมื่อ media ถูกหยุด (pause) ทั้งที่เกิดจากผู้ใช้งานหรือโปรแกรม
onplay script script จะถูก run เมื่อ media พร้อมเล่น
onplaying script script จะถูก run ขณะที่ media กำลังเล่น
onprogress script script จะถูก run เมื่อ browser อยู่ในช่วงกระบวนการได้รับข้อมูลของ media
onratechange script script จะถูก run ทุกครั้งที่ค่าของ playback ถูกเปลี่ยน (เช่น ผู้ใช้งานทำการ slow motion หรือ fast forward)
onseeked script script จะถูก run เมื่อพบว่า attribute ถูกตั้งค่าผิดพลาด
onseeking script script จะถูก run ขณะที่พบว่า attribute นั้นทำงานถูกต้อง
onstalled script script จะถูก run เมื่อ browser ไม่สามารถดึงข้อมูลจาก media ได้(ไม่ว่าจะเกิดจากอะไรก็ตาม)
onsuspend script script จะถูก run ขณะที่การดึกข้อมูลของ media หยุดเนื่องจากการ download นั้นเสร็จสิน (ไม่ว่าจะด้วยอะไรก็ตาม)
ontimeupdate script script จะถูก run เมื่อจุดเริ่มการเล่นของ media ถูกเปลี่ยน (เช่น เกิดจากการที่ผู้ใช้งาน fast toward ในจุดต่างๆของ media )
onvolumechange script script จะถูก run ทุกครั้งเมื่อมีการเปลี่ยนระดับเสียง (volume) รวมไปถึงการตั้งค่าเป็น mute
onwaiting script script จะถูก run เมื่อ media ถูก pause แต่เพื่อการทำงานต่อของ media ในบางส่วน (เช่น การ pause media เพื่อการดึงข้อมูล (buffer) ของ media )

Misc event

Misc Events attribute value คำอธิบาย
ontoggle script เมื่อผู้ใช้งานเปิดและปิด element ของ < details>

2.Differentiation

tag ความเหมือน ความแตกต่าง
< hr>&< br> คือ tag หน้าที่หลักคือการเว้นบรรทัด br คือ การเว้นบรรทัด
hr คือ การขีดเส้นบรรทัดเพื่อเว้นบรรทัด
< ul >&< ol > คือ tag การทำหัวข้อเป็นหัวข้อย้อยๆ ใช้ tag < li >< /li > ทั้งคู่ในการใช้งาน ul คือ การทำหัวข้อย่อยโดยใช้ bullet เป็นหัวข้อ ol คือ การทำหัวข้อย่อยโดยใช้ลำดับในแบบต่างๆเป็นหัวข้อ (เช่น 1,2,3,... A,B,C,...)
< select >&< checkbox >&< radio button > คือการเลือกตัวเลือกคำตอบภายในแบบ form tag ทั้งหมดทำงานภายใน form Pattern ในการเลือกคำตอบของแต่ละ tag จะต่างกันไป
< select > คือ แทปเลือกคำตอบ
< checkbox > คือ Pattern สำหรับการเลือกคำตอบแบบเป็นกล่องเพื่อติ๊กเลือกคำตอบ
< radio button > คือ Pattern สำหรับการเลือกคำตอบแบบปุ่ม bullet
< div >&< span > ใช้ในการนำเสนอข้อความ(content) บน web page สามารถใช้ global & event attribute ได้ทั้งคู่ Pattern การเรียงบน web page แตกต่างกัน
< div > โดยทั่วๆไปจะเรียงในแนวดิ่ง (บนลงล่าง) : block-line
< span > โดยทั่วๆไปจะเรียงในแนวนอน (ซ้ายไปขวา) : inline
< script >& < style > ทั้งคู๋มีหน้าที่หลักในการกำหนดหรือปรับแต่งรูปลักษณ์หรือคุณสมบัติให้ทำงานตาม function ที่ถูกออกแบบไว้โดยจะดูว่า function ต่อไปจะต้องทำอะไร style (CSS) ทำงานโดยแสดง function มาจาก CSS
script (Java script) ทำงานโดยดึงข้อมูลหรือส่งข้อมูลเพื่อให้เกิด function โดยการดึงข้อมูลมาจาก Jquery หรือ Java script (ยังขาดความรู้ตรง Jquery และ Java script)

3.input type

type คำอธิบาย (วิธีใช้งาน)
button ใช้ระบุเพื่อสร้างปุ่มที่สามารถคลิกได้
checkbox ใช้สร้างกล่องเพื่อให้ผู้ใช้งานเลือก
color ใช้ในการระบุสี
date ใช้ในการระบุข้อมูลเรื่องวันที่ : ปี,เดือน,วัน (ไม่มีเวลา)
datetime-local ใช้ในการระบุข้อมูลเรื่องวันที่ ปี,เดือน,วัน และเวลาโดย เวลาจะขึ้นอยู่กับ time zone
e-mail ใช้ระบุพื้นที่ที่ใช้ใส่ e-mail address
file ระบุพื้นที่เปุ่มพื่อให้เลือกไฟล์ (Browse) :ใช้ในการ upload ไฟล์
hidden ใช้ซ้อนเพื่อนที่กรอกข้อมูล
image ใช้ระบุให้รูปภาพมีคุณสมบัติเหมือนปุ่ม submit
month ระบุเดือนและปี
number ใช้ระบุพื้นที่เพื่อใช้ในการใส่ตัวเลข
password ใช้กำหนดพื้นที่สำหรับใส่ password
radio ใช้กำหนดพื้นที่สำหรับให้ผู้ใช้งานเลือกตัวเลือก โดย Pattern ตัวเลือกจะเป็นตัวเลือกแบบ bullet
range ใช้กำหนดขอบเขตของค่าของข้อมูลที่ใช้
reset ใช้ระบุปุ่มใช้ในการ reset
search ใช้ระบุพื้นที่ใส่ข้อความเพื่อใช้ใน string ของ search
submit ใช้กำหนดปุ่ม submit
tel ใช้กำหนดพื้นที่สำหรับใส่เบอร์โทรศัพท์
text ใช้กำหนดพื้นที่สำหรับใส่ ข้อความ (text) : ค่า default ถูกตั้งไว้ 1 บรรทัด
time ใช้ใส่เพื่อระบุเวลา (ไม่เกี่ยวข้องกับ timezone)
url ระบุพื้นที่เพื่อใช้ใส่ URL
week ใช้ระบุสัปดาห์และปี (ไม่เกี่ยวข้องกับ timezone)
